# Scoring Rubric

Two scores per prospect, each 0-5. Both require the evidence gathered in the
research step -- neither is a vibe check.

## Fit score (0-5)

How well does this prospect match the ICP, based on public evidence?

| Score | Criteria |
|---|---|
| 5 | Matches primary ICP role/company profile exactly, has explicit pain or demand evidence, plausible budget/authority to adopt |
| 4 | Matches primary ICP closely, has some evidence, minor uncertainty on authority or budget |
| 3 | Matches adjacent ICP, or primary ICP with weaker/indirect evidence |
| 2 | Plausible fit on firmographics alone, evidence is thin or inferred rather than explicit |
| 0-1 | Should have been filtered by a disqualifier -- don't include these; this row exists only so you notice if one slips through |

## Timing score (0-5)

How likely is "now" to be a good moment, based on public evidence?

| Score | Criteria |
|---|---|
| 5 | Explicit recent trigger (funding, hiring push, deadline, competitor churn) within the domain's relevant recency window |
| 4 | Recent pain/demand signal (days to a couple months old depending on how fast-moving the space is) without an explicit trigger event |
| 3 | Signal exists but is older, or trigger event exists but signal is indirect |
| 2 | Pattern of ongoing pain with no specific recent event -- plausible but not urgent |
| 0-1 | Signal is stale enough that it may no longer reflect their current situation -- flag explicitly if included at all |

## Combining scores

- **Highest-confidence prospect**: fit 4-5 AND timing 4-5, with at least two
  independent pieces of evidence if possible.
- **Shortlist inclusion threshold**: fit >= 3 AND timing >= 2. Below that,
  exclude rather than pad the list.
- Report both scores separately in the table, not just a blended number --
  the user needs to see *why* something scored the way it did (great fit but
  bad timing calls for a different outreach approach than great timing but
  marginal fit).

## Evidence requirement (non-negotiable)

Every prospect scored fit >= 3 needs at least one specific, dated, linked
piece of public evidence attached to the score. If you can't point to it,
the fit score is an assumption, not a score -- exclude the prospect or
explicitly label it as "unverified / low-evidence" rather than presenting it
alongside evidence-backed prospects at the same confidence level.
